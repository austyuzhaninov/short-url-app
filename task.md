# Тестовое задание: Сервис сокращения ссылок (URL Shortener)

## Контекст

Реализовать HTTP-сервис для сокращения ссылок с поддержкой статистики переходов.  
Сервис должен работать с in-memory хранилищем, сохранять данные в файл при остановке и восстанавливать при запуске.

## Функциональные требования

### API

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/shorten` | Принять JSON `{"url": "...", "user_id": "..."}`. Вернуть `{"short_code": "...", "short_url": "..."}`. |
| GET | `/{short_code}` | Перенаправить на оригинальный URL (HTTP 301). |
| GET | `/stats/{short_code}` | Вернуть статистику: оригинальный URL, количество переходов, дату создания. |

### Генерация короткого кода

- Длина — 6 символов (буквы латиницы A-Z, a-z + цифры 0-9)
- При коллизии сгенерировать новый код

### Хранение данных

- Только in-memory хранилище с использованием `sync.RWMutex`
- При перезапуске сервиса данные должны загружаться из файла (формат JSON)
- При штатном завершении работы данные должны сохраняться в тот же файл

## Технические требования

### Обязательные

1. **Чистая архитектура** — разделение на пакеты: handlers, storage, models, config
2. **Обработка ошибок**:
    - Невалидный URL → `400 Bad Request`
    - Несуществующий short_code → `404 Not Found`
    - Внутренняя ошибка → `500`
3. **Конкурентность** — отсутствие data race (проверка через `go test -race`)
4. **Логирование** каждого запроса в формате: `[INFO] 2024-01-15T10:00:00Z POST /shorten 201 duration_ms=12`
5. **Конфигурация через окружение** (env или флаги):
- `PORT` — порт сервера
- `STORAGE_FILE` — путь к файлу хранилища
- `BASE_URL` — базовый URL для коротких ссылок
6. **Graceful shutdown** — корректное завершение с сохранением данных

### Тесты

- Интеграционный тест для `POST /shorten`
- Тест генератора кодов на уникальность в конкурентной среде
- Использовать table-driven tests

## Дополнительные задания (по желанию)

Выполнение этих пунктов повышает оценку, но не является обязательным:

- Middleware для gzip-сжатия ответов
- Rate limiting (10 запросов/сек с одного IP)
- Поддержка кастомных алиасов: поле `custom_code` в запросе `/shorten`
- Удаление ссылок: `DELETE /{short_code}` с проверкой владельца по `user_id`
- Dockerfile + docker-compose.yml

## Примеры запросов

### Создание короткой ссылки

```bash
curl -X POST http://localhost:8080/shorten \
-H "Content-Type: application/json" \
-d '{"url": "https://golang.org/doc", "user_id": "alice"}'
```

#### Ответ:

```json
{"short_code":"aB3xY9","short_url":"http://localhost:8080/aB3xY9"}
```

### Переход по короткой ссылке

```bash
curl -v http://localhost:8080/aB3xY9
```
#### Ответ: HTTP 301 с заголовком Location: https://golang.org/doc

### Получение статистики
```bash
curl http://localhost:8080/stats/aB3xY9
```
#### Ответ:
```json
{"original_url":"https://golang.org/doc","clicks":1,"created_at":"2024-01-15T10:00:00Z"}
```
